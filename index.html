Overview HTML e CSS (Bootstrap)

Conceitos básicos para boas práticas de estruturação e estilo

Identação de código

A estrutura HTML é a base para o desenvolvimento de qualquer aplicação web. A semantica do HTML deve ser simples e concisa, tendo como objetivo estruturar a base da aplicação com o mínimo de elementos e o máximo de desempenho.

Hierarquia dos Elementos

Por se tratar da estruturação básica da aplicação, o HTML precisa dar uma visão simples e objetiva de como os elementos estão hierarquizados.

A hierarquia é fundamental no HTML, uma vez que precisamos ter bem definidos quais são os "irmãos", "filhos" e "pais"

< Tags />

Tags são os <html>, <body>, <div> enquanto os elementos são a parte semântica que as tags criam.

Atributos

Atributos são informações que passamos para os elementos para que seu comportamento e funcionalidades sejam aplicados.

Alguns elementos podem ter diversos atributos, que são capazes de mudar desde seu estilo (CSS) até suas informações e a própria existência do elemento.

Exemplo:
<input placeholder= "Planning infinita" value= "Buzz Lightyear" >

<div class= "form-control" value= "Buzz Lightyear" >

Conceitos de grid (Elementos inline e block)

Muitas vezes isso não é explicado, mas os elementos HTML são dividos em duas classes: Os elementos INLINE e os elementos BLOCK

O conceito de grid exemplifica como os elementos se organizam no documento.

Visualizando a imagem podemos ver como os elementos se organizam de forma semelhante a um grid de larga de uma corrida.

Alinhamento dos elementos

Importante lembrar que os elementos do HTML naturalmente se alinham para o lado superior esquerdo da exibição, portanto lemos o código como se realmente fosse um texto.

Inline vs Block

Conforme vimos na parte de hierarquia do HTML, o elemento pode ter diversas relações (ter pais, irmãos etc) e sua classificação como inline ou como block, define como será seu comportamento padrão com seus familiares. Enquanto um elemento block ocupa todo o espaço do seu elemento pai, empurrando os irmãos para a linha abaixo, os elementos inline agrupam-se na mesma linha, formando um alinhamento para o canto superior esquerdo.

Ocupam somente o espaço horizontal necessário, alinhando todos os elementos à esquerda.

Ocupam todo espaço horizontal disponível pelo elemento pai, empurrando todos os elementos à direita.

A estrutura inline e block é definida de forma nativa pelo HTML5, porém veremos mais a frente como o CSS altera essa estrutura com algumas condições.
